# 画面仕様書

## 1. 文書情報

- 対象システム: Quiz Streaming App
- 対象実装: 2026-07-17 時点のリポジトリ
- 本番URL: `https://quiz-streaming-app.kokage-studio.com`
- クライアント: Nuxt 4 / Vue 3 のSPA
- リアルタイム同期: Firebase Realtime Database

本書は現在の画面実装を記述する。将来構想ではなく、コード上で動作する内容を仕様とする。

## 2. 画面一覧

| 画面ID | 画面名 | パス | 利用者 | 保護 |
|---|---|---|---|---|
| SCR-001 | トップ・ルーム参加 | `/` | 参加者・管理者 | 公開 |
| SCR-002 | 参加者用クイズルーム | `/room/:roomId` | 参加者 | 公開 |
| SCR-003 | 管理ルーム一覧 | `/admin` | 管理者 | Cloudflare Access + 管理API認可 |
| SCR-004 | ルーム作成・編集 | `/admin/edit` | 管理者 | Cloudflare Access + 管理API認可 |

## 3. 共通仕様

### 3.1 ルームID

- 使用可能文字は半角英数字と `.`, `_`, `~`, `-`。
- 正規表現は `^[A-Za-z0-9._~-]+$`。
- 大文字と小文字は区別される。
- URLに埋め込むときはURLエンコードする。

### 3.2 ルーム情報の取得

画面は `GET /api/rooms/:roomId` からルーム設定を取得する。D1に存在しない場合は、静的ファイル `/data/rooms/:roomId.json` をフォールバックとして参照する。

### 3.3 リアルタイム接続表示

Firebase接続状態は次の値を取る。

| 状態 | 意味 | 画面表示 |
|---|---|---|
| `idle` | 未接続 | 状態値を表示 |
| `connecting` | 接続中 | 状態値を表示 |
| `connected` | 接続済み | 「接続中」 |
| `error` | 接続失敗 | エラー通知 |
| `unconfigured` | Firebase環境変数なし | 設定不足の通知 |

## 4. SCR-001 トップ・ルーム参加

### 4.1 目的

案内されたルームIDを入力し、参加者用ルームへ遷移する。管理画面への入口も提供する。

### 4.2 表示要素

- ブランド名 `QUIZ STREAM`
- ルームID入力欄
- 「参加する」ボタン
- ハンバーガーメニュー
  - 「既存ルームを管理する」: `/admin`
  - 「ルームを作る」: `/admin/edit`

### 4.3 入力・遷移

- ルームIDが空、または使用可能文字以外を含む間は「参加する」を無効化する。
- 不正な文字がある場合は「英数字と . _ ~ - のみ使用できます」と表示する。
- 送信すると `/room/{roomId}` へ遷移する。

## 5. SCR-002 参加者用クイズルーム

### 5.1 目的

管理者の進行に同期してスライドを閲覧し、クイズへ回答する。

### 5.2 初期処理

1. URLから `roomId` を取得する。
2. ルーム設定をAPIから取得する。
3. Firebaseの `rooms/{roomId}/runtime` を購読する。
4. ローカルに現在のセッション用ニックネームがなければ、ニックネーム入力モーダルを表示する。

### 5.3 表示要素

- ルーム名、ルームID、ニックネーム
- 現在のスライド画像
- スライド番号、スライドID、進行モード、同期状態
- 現在のスライドが問題の場合
  - 問題番号、問題文
  - 単一選択または複数選択の選択肢
  - 残り時間
  - 回答ボタンと回答状態
- 優勝者発表モーダル
- 接続エラー通知

### 5.4 ニックネーム

- 必須、最大30文字。
- 前後の空白を除去し、連続する空白を1文字に正規化する。
- `localStorage` にルーム単位で保存する。
- 管理者がセッションをリセットすると、参加者ID、ニックネーム、ローカル回答履歴を削除して再入力を求める。

### 5.5 回答状態

| 状態 | 条件 | 操作 |
|---|---|---|
| 待機中 | `questionOpen=false` かつ締切・解答・結果モードでない | 回答不可 |
| 受付中 | `mode=question` かつ `questionOpen=true` | 選択・送信可能 |
| 締切済み | `mode` が `closed`, `answer`, `result` のいずれか | 回答不可 |
| 回答済み | 同一端末・同一セッション・同一問題の回答履歴あり | 再回答不可 |

### 5.6 タイマーと自動送信

- 回答受付開始時のFirebaseサーバー時刻を基準に、各端末で残り時間を算出する。
- 表示更新間隔は250ミリ秒、表示値は秒単位で切り上げる。
- 残り5秒以下で緊急表示、0秒で終了表示にする。
- 0秒到達時に未送信の選択肢がある場合、その選択内容を1回だけ自動送信する。
- 未選択の場合は自動送信しない。

### 5.7 回答送信

- Firebaseの `rooms/{roomId}/answers/{questionId}/{participantId}` へ保存する。
- 同一参加者ID・同一問題のデータは `set` により上書きされる。
- 画面側はローカル回答履歴によって通常の再送を防止する。

### 5.8 エラー表示

- ルーム取得失敗時: 「部屋情報を読み込めませんでした」と対象roomIdを表示する。
- Firebase未設定・接続失敗時: 「リアルタイム同期に接続できません」と詳細を表示する。
- 回答送信失敗時: Firebaseから得たエラーメッセージを問題パネル内に表示する。

## 6. SCR-003 管理ルーム一覧

### 6.1 目的

ログイン中の管理者が管理可能なルームを一覧表示し、配信または編集画面へ遷移する。

### 6.2 認証状態

画面表示時に `GET /api/admin/session` を呼び出す。

| 状態 | 表示 |
|---|---|
| 確認中 | ローダーと「認証情報を確認しています」 |
| 認可済み | ルーム一覧 |
| 403 | アクセス拒否モーダル。3秒後に `/` へ遷移 |
| その他エラー | 現行実装では専用表示なし |

### 6.3 一覧内容

- ログインメールアドレス
- システム管理者の場合は「システム管理者」表示
- ルームID、ルーム名、更新日時
- 「配信」: `/admin/room/{roomId}`
- 「編集」: `/admin/edit?q={roomId}`
- 「ルームを作る」: `/admin/edit`

一覧が空の場合は空状態メッセージと作成ボタンを表示する。

## 7. SCR-004 ルーム作成・編集

### 7.1 モード

| モード | URL | 保存API |
|---|---|---|
| 新規作成 | `/admin/edit` | `POST /api/admin/rooms` |
| 編集 | `/admin/edit?q={roomId}` | `PATCH /api/admin/rooms/{originalRoomId}` |

`roomId` クエリも読み込み用として受け付けるが、画面自身は保存後に `q` を使用する。

### 7.2 基本情報

- ルームID: 必須、共通のルームID文字制約に従う。
- ルーム名: 必須。保存時に前後空白を除去する。
- `description` はデータ型には存在するが、現行画面に編集欄はない。
- 保存時の `initialSlideIndex` は常に `0` に正規化する。

### 7.3 コンテンツ編集

- 画像をファイル選択またはドラッグ＆ドロップで追加する。
- キャンバスのダブルクリックまたはボタンからクイズを追加する。
- コンテンツをドラッグして順序変更する。
- チェックした複数コンテンツを一括削除する。
- 右クリックメニューから画像追加、クイズ追加、対象コンテンツ削除を行う。

### 7.4 画像コンテンツ

- `image/*` のファイルのみアップロード対象とする。
- 追加前に有効なルームIDが必要。
- R2アップロード成功後、スライド末尾に追加する。
- 画像タイトルを編集できる。
- 画像の差し替え時、以前の画像が同一ルーム配下のR2オブジェクトなら削除する。

### 7.5 クイズコンテンツ

- 新規クイズの初期値
  - 種別: 単一選択
  - 選択肢: 2件
  - 制限時間: 20秒
- 種別は単一選択または複数選択。
- 選択肢は2～6件。フォーカス離脱時、3件目以降の空欄を除去してラベルを `A` から振り直す。
- 単一選択は正解がちょうど1件必要。
- 複数選択は正解が1件以上必要。
- 問題文、各選択肢、正解、1秒以上の制限時間が必須。
- クイズ画像は任意。追加、差し替え、削除が可能。

### 7.6 保存

- 必須条件を満たすまで保存ボタンを無効化する。
- 保存処理は最大5回試行する。
- 保存後に編集がある場合は、変更から1秒後に自動保存する。
- 状態表示は「未保存」「保存中」「保存済」。
- ルームID変更もPATCHで処理し、D1上では新IDへの挿入と旧IDの削除を行う。
- ルームID変更時、既存のR2画像キーやFirebaseデータは自動移行されない。

### 7.7 共有URL

初回保存後、現在のオリジンを使用して次のURLとQRコードを生成する。

- 参加者用: `/room/{roomId}`
- 管理者用配信画面: `/admin/room/{roomId}`

URLコピーとPNG形式のQRコード保存ができる。

## 8. SCR-005 管理者用配信操作

### 8.1 目的

参加者画面と同期するスライド進行、回答受付、集計、優勝者発表を操作する。

### 8.2 表示要素

- ルーム名、ルームID、現在のスライド番号・ID
- 現在スライドのプレビュー
- 進行状態、回答受付状態、対象問題ID
- 回答受付タイマー
- スライド操作ボタン
- 問題文、選択肢、正解表示
- 選択肢別集計、ニックネーム別回答
- ニックネーム別成績
- 優勝者発表操作
- セッションリセット操作

### 8.3 スライド操作

| 操作 | 結果 |
|---|---|
| 最初へ / 前へ / 次へ / 最後へ | 対象スライドへ移動し、`mode=slide`, `questionOpen=false` にする |
| 回答受付開始 | 問題スライドで `mode=question`, `questionOpen=true`。開始時刻をサーバー時刻で記録 |
| 回答締切 | `mode=closed`, `questionOpen=false` |
| 正解表示 | `mode=answer`, `questionOpen=false` |
| 集計表示 | `mode=result`, `questionOpen=false` |

Firebaseが接続済みでない場合、操作ボタンを無効化する。

### 8.4 回答集計

- 現在の `sessionId` と一致する回答のみを対象とする。
- 同じニックネームの回答が複数ある場合、問題別表示では最新回答を採用する。
- 選択肢ごとの選択人数と割合を表示する。複数選択では人数合計が回答者数を超える場合がある。

### 8.5 成績と優勝者

- 正解は正解選択肢集合と回答選択肢集合の完全一致で判定する。
- 1問正解につき1点。
- ニックネーム単位で集計し、同一問題は最初に処理した1回答のみを採点する。
- 並び順は得点降順、回答問題数降順、ニックネーム昇順。
- 最終スライドのみ「GO」を表示する。
- 最高得点が同じ参加者は全員を同率優勝として発表する。

### 8.6 セッションリセット

- 確認ダイアログで承認後、新しいUUIDを `sessionId` に設定する。
- スライドを `initialSlideIndex` に戻し、回答受付、開始時刻、優勝者表示をクリアする。
- Firebase上の過去回答は削除しない。新しい `sessionId` によるフィルタで集計対象外にする。
- 参加者端末はセッション変更を検知し、ローカルの参加者情報と回答済み状態を削除する。

## 9. 進行モード

| 値 | 意味 |
|---|---|
| `slide` | 通常スライド表示 |
| `question` | 回答受付中 |
| `closed` | 回答締切後 |
| `answer` | 正解表示 |
| `result` | 集計表示 |

## 10. 現行制約

- 管理者用配信画面自身は管理セッションAPIを呼ばず、Cloudflare Accessによる `/admin/*` 保護を前提とする。
- Realtime Database Rules上、`runtime` は認証なしで書き込み可能。管理者URLの秘匿だけでは不正操作を防げない。
- 参加者の再回答防止は端末の `localStorage` に依存し、別ブラウザ・ストレージ削除・直接Firebase操作を防げない。
- ニックネームが同じ参加者は成績上同一人物として集約される。
- ルーム削除UIおよび削除APIはない。
